# 数据分析技能

## 技能说明

分析结构化数据（CSV/JSON），生成统计报告、趋势分析和可视化建议。

## 使用场景

| 场景 | 说明 |
|------|------|
| 销售数据 | 分析销售趋势，识别高峰低谷 |
| 用户行为 | 分析用户活跃度、留存率 |
| A/B 测试 | 分析测试结果，给出显著性结论 |
| 财务数据 | 分析收支情况，预测趋势 |
| 运营指标 | 分析 KPI 变化，找出异常 |

## 工作流程

### Step 1: 数据加载与验证

读取用户提供的数据文件并验证：

- 检查文件格式（CSV/JSON）
- 识别列名和数据类型
- 检查数据完整性（缺失值、空行）
- 确认数据规模（行数、列数）

### Step 2: 数据清洗

处理常见数据问题：

**缺失值处理：**
- 数值列：使用均值/中位数填充，或标记为缺失
- 分类列：使用众数填充，或标记为"未知"
- 时间列：前向/后向填充

**异常值检测：**
- 使用 IQR 方法检测数值异常值
- 使用 Z-score 方法检测极端值
- 列出异常值供用户确认

**格式统一：**
- 日期格式标准化
- 数值格式统一（千分位、小数位）
- 分类大小写统一

### Step 3: 统计分析

计算统计量和模式识别：

**描述性统计：**
- 均值、中位数、标准差
- 最小值、最大值、四分位数
- 频数分布（分类变量）

**趋势分析：**
- 时间序列趋势（如适用）
- 环比/同比增长
- 移动平均平滑

**相关性分析：**
- 变量间相关系数
- 强相关变量配对
- 多重共线性检查

### Step 4: 生成报告

输出结构化分析报告：

```
数据概览
- 数据规模：N 行 × M 列
- 时间范围：...
- 数据完整性：...

关键发现
1. [发现1] - [数据支持]
2. [发现2] - [数据支持]
3. [发现3] - [数据支持]

趋势分析
- [趋势1]：...
- [趋势2]：...

可视化建议
- [图表1]：[类型] - [目的]
- [图表2]：[类型] - [目的]

行动建议
1. [建议1]
2. [建议2]
```

## 注意事项

- 数据量限制在 100 万行以内（超出需采样）
- CSV 文件需使用 UTF-8 编码
- JSON 文件需为标准格式（数组或对象）
- 复杂分析需要明确指定分析目标
- 预测性分析需要额外说明

## 示例

**输入：**
```
分析这个销售数据文件：sales_2025.csv
我想了解销售趋势和最佳销售渠道。
```

**输出：**
```
📊 销售数据分析报告

## 数据概览
- 文件：sales_2025.csv
- 规模：1,250 行 × 8 列
- 时间范围：2025-01-01 至 2025-12-31
- 完整性：98.5%

## 关键发现

1. **Q4 销售额显著增长**
   - 数据：Q4 平均日销 ¥50,000 vs Q1 ¥30,000
   - 解读：年终促销效果明显，建议加强 Q4 投入

2. **线上渠道超过线下**
   - 数据：线上占比 52%（2024: 35%）
   - 解读：渠道转移趋势确立，优化线上投入

3. **华东地区贡献最高**
   - 数据：华东占全国 38%，同比增长 25%
   - 解读：华东市场成熟，可作为标杆推广

## 趋势分析

- 整体趋势：上升 (+15% 同比)
- 季节性：Q4 最高，Q2 最低
- 渠道趋势：线上持续增长，线下平稳

## 可视化建议

1. 折线图 - 月度销售额趋势
2. 堆叠柱状图 - 各渠道销售占比变化
3. 热力图 - 各地区销售热力分布
4. 饼图 - 当前渠道市场份额

## 行动建议

1. Q4 提前 2 个月准备促销库存
2. 将华东成功案例复制到华南
3. 线上渠道追加 20% 广告预算
4. 调查 Q2 销售低谷原因
```
